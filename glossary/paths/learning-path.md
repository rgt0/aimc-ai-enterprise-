# AIMC Learning Path – From CLI to Swarm Thinking

This document defines a structured learning path for understanding
AI systems in an enterprise context.

It builds from execution basics to system-level behavior.

---

## Phase 1 – Runtime & Execution Basics

Goal: remove "magic thinking"

- CLI (Command Line Interface)
- npm / npx
- Execution model
- Hooks

Key insight:
AI does not exist outside runtime environments.

---

## Phase 2 – Components

Goal: understand building blocks

- Large Language Model (LLM)
- Agent
- AI System

Key insight:
An LLM is not the system.

---

## Phase 3 – Agentic Layer

Goal: understand behavior evolution

- Agent
- Agentic AI
- Multi-Agent System

Key insight:
AI evolves from response → action → coordination

---

## Phase 4 – System Behavior

Goal: understand real-world behavior

- Swarm Behavior
- Non-determinism
- Context Drift

Key insight:
AI systems behave, not just respond.

---

## Phase 5 – Knowledge Layer

Goal: understand information flow

- RAG
- Context
- Document
- Chunk

Key insight:
AI output quality = knowledge + context

---

## Phase 6 – Platform Layer

Goal: understand enterprise constraints

- STRIVE Platform
- AI Operator (AIO)
- AI Engineer (AIE)

Key insight:
AI capabilities are shaped by platform constraints.

---

## Phase 7 – Legacy Integration

Goal: understand real-world complexity

- Legacy Systems
- COBOL

Key insight:
AI adds value where complexity already exists.

---

## Phase 8 – Governance

Goal: understand control

- Governance
- Responsibility
- Human-in-the-Loop

Key insight:
AI does not carry accountability.

---

## Phase 9 – External Systems

Goal: make behavior observable

- Ruflo

Key insight:
External systems help visualize internal AI behavior.

---

## Final Insight

Understanding enterprise AI requires moving through:

Execution → Components → Behavior → Knowledge → Platform → Governance

---

## One-Sentence Summary

AI is not a tool — it is a layered system that must be understood and controlled step by step.
