# AIMC – AI in the Enterprise

This repository is part of the **AIMC (AI Mindset & Competence)** initiative.

Its purpose is to build **system-level understanding of enterprise AI**  
in the context of **T‑Systems / Deutsche Telekom**, with a strong focus on:

- real-world constraints
- governance
- platform integration
- legacy modernization
- responsible AI usage

This is **not** a product manual.  
This is **not** a user guide.  
This is an **operating model and thinking framework**.

---

## 🎯 Why This Repository Exists

Most AI materials focus on:
- prompting
- productivity hacks
- tool features
- demos

In enterprise environments, this is not enough.

What actually determines success or failure is:
- integration
- IAM
- governance
- platform dependencies
- human validation
- legacy systems

This repository exists to answer a different question:

> **How does AI really work inside an enterprise system – and where does it break?**

---

## 🧠 What This Repository Focuses On

- AI systems **as systems**, not as “smart tools”
- AI Operator (AIO) and AI Engineer (AIE) as **platform components**
- STRIVE as an **enterprise execution platform**
- RAG-based systems and their limitations
- Knowledge, context, and document reality
- COBOL and legacy modernization with AI
- Human responsibility and accountability
- The gap between **ideal AI narratives** and **enterprise reality**

## 🧩 Example: Ideal vs Enterprise Reality

**Ideal (demos, marketing):**

Human → AI → Result

**Enterprise reality:**

Human
→ AI System (AIO / AIE)
→ STRIVE Platform
→ IAM & Provisioning
→ Integrations
→ Governance
→ Infrastructure
→ LLM
→ back through the system
→ Result

This repository explains **why this matters**.

---

## 🧠 AI Operator & AI Engineer Perspective

- AI Operator and AI Engineer are **systems**, not users
- They are **RAG-based, governed platforms**
- They do not “know documents” — they process **retrieved fragments**
- Output quality depends on:
  - document structure
  - context selection
  - governance constraints
  - human validation

This repository explains these behaviors and their consequences.

---

## 🧱 Legacy & COBOL Context

COBOL and legacy systems are treated as **first-class citizens** here.

Key principle:

AI does not replace legacy understanding.
AI accelerates the loop:
understanding → correction → refinement.

Modernization is presented as an **iterative sensemaking process**, not a one-click transformation.
