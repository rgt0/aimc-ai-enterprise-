# Code-Centric vs System-Centric AI Mentoring

This document describes two complementary approaches to AI-supported mentoring
in COBOL / Mainframe environments.

It is not tool-specific and focuses on **conceptual roles** within
enterprise AI mentoring (AIMC).

---

## 🎯 Why This Distinction Matters

COBOL mentoring faces a fundamental challenge:
Understanding is required at two different levels:

Code level (what does the program do?)
System level (why does the system behave this way?)


AI tools tend to focus on one level — effective mentoring requires both.

---

## 🟦 Code-Centric AI Mentoring

### Definition

Code-centric mentoring focuses on understanding:

- program structure
- control flow
- statements and variables
- technical behavior of the code

---

### Typical Capabilities

- code viewer
- summary explanation of programs
- line-by-line explanation
- lesson mode / guided walkthrough

---

### Strengths

- helps juniors understand unfamiliar syntax
- accelerates onboarding
- makes implicit technical logic visible

---

### Limitations

- lacks full system context
- may misinterpret business intent
- cannot distinguish design from workaround
- may create false confidence

---

## 🟧 System-Centric AI Mentoring (AIMC / AIO)

### Definition

System-centric mentoring focuses on understanding:

- system purpose
- business logic distribution
- architectural constraints
- operational risks

---

### Typical Capabilities

- explaining system-level behavior
- identifying patterns and risk zones
- connecting technical elements to business meaning
- highlighting governance and constraints

---

### Strengths

- provides context before code analysis
- supports decision-making preparation
- reduces risk of misinterpretation
- aligns mentor and junior mental models

---

### Limitations

- less precise at code-level details
- requires curated knowledge base
- depends on human validation

---

## 🔁 Combined Mentoring Flow

Effective COBOL mentoring combines both approaches:


System understanding (AIO)
↓
Shared mental model
↓
Code-level exploration
↓
Human validation

---

## 🧩 Conceptual Flow


COBOL System Landscape
↓
System-Centric AI (AIO)
↓
Context & Framing
↓
Code-Centric AI Tool
↓
Detailed Understanding
↓
Human Validation

---

## ❗ Key Principle


Code analysis without system context is unsafe.
System understanding without code validation is incomplete.

---

## 🧠 AIMC Perspective

From an AIMC perspective:

- AI tools should **support cognition**, not replace judgment
- mentoring should **alternate between abstraction levels**
- responsibility must remain with humans at all times

---

## 🔮 Future Direction

In enterprise environments, these approaches will converge:

- system-level AI defines boundaries and context
- code-level AI provides detailed insight
- human experts validate and decide

---

## ✅ Summary

- Code-centric AI explains *how*
- System-centric AI explains *why*
- Effective mentoring requires both
- AI augments mentoring, not replaces it
